2.3 Permissions & Ownership

Every file has:

  • Owner: User who owns the file
  • Group: Group ownership
  • Permissions: Read (r), Write (w), Execute (x)

Permission Format

-rwxr-xr--
‚îÇ‚îÇ‚îÇ‚îÇ‚îÇ‚îÇ‚îÇ‚îÇ‚îÇ
‚îÇ‚îî‚î¥‚î¥‚î¥‚î¥‚î¥‚î¥‚î¥‚î¥ Permissions
‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ File type (- = file, d = directory)

Owner: rwx (7)
Group: r-x (5)
Other: r-- (4)
Result: 754

Changing Permissions

chmod 755 file          # rwxr-xr-x
chmod u+x script.sh     # Add execute for owner
chmod g-w file          # Remove write for group
chmod o+r file          # Add read for others

Changing Ownership

chown user:group file
chown -R user:group /directory

# Example
chown nginx:nginx /var/www/html

umask

Controls default file permissions when files are created.

umask 022               # Default: files 644, dirs 755

3.4 Merging & Rebasing

Merge

# Merge feature into main
git checkout main
git merge feature/login

# Merge with no fast-forward (preserves history)
git merge --no-ff feature/login

Rebase (Clean History)

# Rebase feature on top of main
git checkout feature/login
git rebase main

# Interactive rebase (squash commits)
git rebase -i HEAD~3

# Continue after resolving conflicts
git rebase --continue
git rebase --abort              # Cancel rebase
üí° When to Use:
  • Merge: Preserves complete history, good for main branches
  • Rebase: Creates linear history, good for feature branches before merge

4.9 Debugging Common Issues

Case 1: App Cannot Connect to Database

‚ö†Ô∏è Checklist: 
1. Check Security Group inbound rules
   - Does DB SG allow port 3306 from app SG?

2. Check Security Group outbound rules
   - Does app SG allow outbound to DB port?

3. Check VPC routes
   - Are both in same VPC or peered VPCs?

4. Check DNS resolution
   - Can you resolve DB hostname?
   - nslookup db.example.com

5. Check if port is open
   - telnet db-host 3306
   - nc -zv db-host 3306

6. Check database is running
   - systemctl status mysql

   - /var/log/mysql/error.log

‚ö†Ô∏è Checklist:

   - kubectl exec -it pod -- nslookup kubernetes.default

‚ö†Ô∏è Checklist:

   - dig example.com
   - Does it return correct IP?

2. Check if server is reachable

   - telnet server-ip 80

3. Check web server status
   - systemctl status nginx
   - systemctl status apache2

   - tail -f /var/log/nginx/error.log

5. Check firewall rules

   - iptables -L

6. Check Security Group (if cloud)
   - Is port 80/443 open to 0.0.0.0/0?

6.2 Remote State (MANDATORY for Teams)

‚ö†Ô∏è Critical: Never store state locally in production. Always use remote backend with locking.

    bucket         = "tf-state-prod-bucket"
    key            = "vpc/terraform.tfstate"
    region         = "ap-south-1"
    dynamodb_table = "terraform-state-lock"

  }
}

Create Lock Table:

aws dynamodb create-table \
